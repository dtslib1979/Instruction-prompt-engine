**커밋 컨벤션**:

- `feat|fix|docs|style|refactor|perf|chore`
- 예: `feat(chapter): 모바일 입력 루틴 추가`

**브랜치 전략**:

- `main`: 프로덕션 브랜치
- `develop`: 개발 통합 브랜치
- `feature/기능명`: 기능 개발 브랜치

**PR 제목 규칙**:

```
[TYPE] 간단한 변경사항 설명

- 변경된 주요 내용 1
- 변경된 주요 내용 2
- 테스트 및 검증 방법
```

**코드 리뷰 체크리스트**:

- [ ] 코드 스타일 가이드 준수
- [ ] 테스트 커버리지 확인
- [ ] 문서 업데이트 여부
- [ ] 성능 영향도 검토
